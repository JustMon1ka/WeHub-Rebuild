## 目录结构一览
<pre> 
Services/                      # 后端（C#）项目根目录  
└── UserAuthService/           # 用户认证服务  
    ├── Controllers/           # 路由控制器，用于处理路由请求、转发服务层逻辑  
    ├── Data/                  # 数据上下文类，配置 EF Core 的数据库连接与实体关系。  
    ├── Models/                # 实体模型定义，表示数据库中数据表结构。  
    ├── DTOs/                  # 数据传输对象，用于前后端接口交互建模  
    ├── obj/                   # 编译产物与中间文件（无需修改，自动生成），在gitignore中  
    ├── bin/                   # 编译产物与中间文件（无需修改，自动生成），在gitignore中  
    ├── Properties/            # 启动配置，定义调试参数。  
    ├── Repositories/          # 数据访问层，封装对数据库的具体读写操作。  
    ├── Services/              # 服务层逻辑，用于处理业务核心逻辑（认证、令牌生成等）。  
    ├── appsettings.json       # 该服务的配置文件，比如配置JWT令牌，数据库链接等（涉密文件，千万别上传！！！）  
    ├── template.appsettings.json # appsettings.json的模板文件，为了不泄密的同时明确该文件的配置项内容而创建，使用是删除前缀template然后填入需要的内容
    ├── Program.cs             # 服务程序的主入口，其中需要定义和配置该服务使用到的中间件（身份认证，Swagger UI，数据库服务等）  
    └── UserAuthService.csproj # 该服务（也是解决方案）的依赖项和启动项配置文件  
├── UserDataService/           # 用户数据服务  
├── xxxService/                # 更多待拓展的服务  
└── Shared/                    # 多个服务之间可以共用的一些代码段  
    └── Models/                # 多个服务可以共用的实体模型  
        └── User.cs            # User模型（在数据库中对应USERS表），在UserAuthService和UserDataService中共用  
    └── ......  
└── Tests/                     # 测试单元根目录  
    └── UserAuthService.Tests/ # UserAuthService的测试单元合集  
        └──LoginBy3Methods.cs  # UserAuthService下的一个测试单元，这个测试单元测试的是：注册用户 -> 用户名登录 -> 手机号登录 -> 邮箱登录 -> 注销用户的流程  
    └── UserDataService.Tests/ # UserDataService的测试单元合集  
        └── ......  
</pre>
